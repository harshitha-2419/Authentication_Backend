# Database Schema

## Collection: `users`

| Field | Type | Required | Unique | Default | Description |
|-------|------|----------|--------|---------|-------------|
| `name` | String | ✅ | ❌ | - | Full name of the user |
| `email` | String | ✅ | ✅ | - | Email address (stored lowercase) |
| `phone` | String | ✅ | ✅ | - | 10-digit phone number |
| `password` | String | ✅ | ❌ | - | bcrypt hashed password |
| `isVerified` | Boolean | ❌ | ❌ | `false` | True after OTP email verification |
| `role` | String | ❌ | ❌ | `"user"` | One of: `user`, `admin`, `super_admin` |
| `otp` | String | ❌ | ❌ | `null` | 6-digit OTP code (cleared after use) |
| `otpExpiry` | Date | ❌ | ❌ | `null` | OTP expiration timestamp (5 min) |
| `otpAttempts` | Number | ❌ | ❌ | `0` | Wrong OTP attempts counter (max 3) |
| `otpResendCount` | Number | ❌ | ❌ | `0` | OTP resend counter (max 3) |
| `resetToken` | String | ❌ | ❌ | `null` | SHA-256 hashed password reset token |
| `resetTokenExpiry` | Date | ❌ | ❌ | `null` | Reset token expiry (15 min) |
| `refreshToken` | String | ❌ | ❌ | `null` | JWT refresh token (cleared on logout) |
| `createdAt` | Date | ❌ | ❌ | auto | Auto-generated by Mongoose timestamps |
| `updatedAt` | Date | ❌ | ❌ | auto | Auto-generated by Mongoose timestamps |

---

## Field Details

### `password`
- Never stored as plain text
- Hashed using `bcrypt` with 12 salt rounds
- Example stored value: `$2b$12$...`

### `role`
- `user` — default role, access to own profile only
- `admin` — can view all users
- `super_admin` — can view, delete users and change roles

### `otp` / `otpExpiry`
- Generated on register and login-otp request
- Cleared after successful verification
- Expires in 5 minutes

### `otpAttempts`
- Increments on each wrong OTP entry
- Blocked at 3 — user must resend OTP to reset
- Resets to 0 on successful verify or resend

### `otpResendCount`
- Increments on each resend request
- Blocked at 3 — prevents OTP spam
- Resets to 0 only after successful OTP verification

### `resetToken`
- Raw token sent in email link
- SHA-256 hash stored in DB for security
- Expires in 15 minutes
- Cleared after successful password reset

### `refreshToken`
- Stored in DB on login
- Used to issue new access tokens
- Set to `null` on logout — immediately invalidates it

## Indexes

MongoDB automatically creates indexes for:
- `email` (unique)
- `phone` (unique)
- `_id` (default)
